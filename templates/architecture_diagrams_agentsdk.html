Agent SDK View

How LovelyChaos uses the Agent SDK, explained in plain English.

This is the simple version of the architecture page. LovelyChaos uses a "one brain" model: email and SMS share a single household conversation, so the assistant has full context no matter which channel a parent uses. The Agent SDK pulls events out of messy school updates, routes them, and runs follow-up commands with tools. The app provides the real integrations underneath.

What it does

What the app still does

The app provides the real integrations: calendar writes, reminder scheduling, preference saves, notifications, storage, and audits.

One brain, every channel

Diagram 1

A school message comes in

A school email or text reaches LovelyChaos first. The app saves it, opens any useful links or attachments, and then starts an Agent SDK run. The agent first extracts the event candidates, then a second agent step routes each one into the right outcome. Only after that do the app and summary flow finish the household update.

Diagram 2

A parent replies later

When a parent replies — by email or SMS — LovelyChaos does not start from zero. It loads the unified household session which contains conversation history from both channels, tagged with [via sms] or [via email]. Documents and followup context created from email are visible to SMS and vice versa. The agent resolves the parent's intent, then uses function tools to act on it.

Diagram 3

When function tools are used

Function tools are not used when a school update first comes in. They start only after a later follow-up email or SMS is understood as a supported command. At that point the app builds a safe runtime, the Agent SDK picks the right tool path, and the tool runs the real app code for preferences, calendar lookups, adds, updates, deletes, or reminders.

Diagram 4

One brain across channels

This is what makes LovelyChaos feel like one assistant instead of two disconnected bots. A school newsletter arrives via email, the app extracts events and creates followup context. Later, the parent texts a question about an event from that email. The SMS flow finds the email-originated context via cross-channel lookup, loads recent documents, and replies with full awareness — all from one unified household session.

The simplest mental model

LovelyChaos uses separate Agent SDK passes for extraction, routing, and tool-backed follow-up execution. Email and SMS share one household session, so the assistant is always one brain.

One brain, not two bots

A single session key (household:{id}) stores all conversation turns tagged by channel. Followup context and documents cross channels automatically, so SMS can reference an email digest and vice versa.

Why tracing matters

Each important workflow and tool run can be traced, which makes it easier for the team to inspect what happened when a result looks wrong or surprising.
