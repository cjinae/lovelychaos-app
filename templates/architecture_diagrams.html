System View

How LovelyChaos combines deterministic logic and LLM decisions.

These diagrams map the current code paths in /webhooks/email/inbound, /webhooks/sms/inbound, the content-analysis pipeline, the summary pipeline, and the follow-up command handlers. Deterministic gates handle trust, tenant safety, policy, storage, and side effects. LLM calls handle extraction, summarization, and intent parsing.

Diagram 1

Email ingestion: newsletter in, structured summary and actions out

The app starts with strict deterministic trust checks, then uses the LLM for event extraction and summary synthesis, and finally returns to deterministic policy logic for validation, routing, calendar writes, notifications, and audit.

`/webhooks/email/inbound` `resolve_and_download_links` `build_brief_summary`

Diagram 2

Reply handling: email and SMS follow-up commands

Replies are lighter on model work. The LLM mainly classifies intent and sometimes re-extracts an event from forwarded content. Deterministic logic owns disambiguation, follow-up context lookup, tenant safety, calendar mutations, reminders, preference writes, and response delivery.

`/webhooks/sms/inbound` `parse_command()` `_handle_add_command()`

Why the split is useful

Across both diagrams, the model is used for interpretation and compression, not for trust or mutation safety. All side effects stay behind deterministic gates.

Where context lives

`FollowupContext` and `SmsConversationState` are the connective tissue that let a short follow-up like “add that one” or “tell me more about swim” resolve without re-reading the whole thread manually.
